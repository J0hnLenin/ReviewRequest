# ReviewRequest
Service for assigning a code verification request to a team member

# Принятые решения

## Один User может быть только в одной команде одновременно

*Что если в сервис придёт запрос 1 который содержит пользователя u1 и обновляет команду t1,  
а затем запрос 2, который содержит пользователя u1 и обновляет команду t2?

Можно поступить двумя способами:
1. поддерживать логику 1 User -> many Team
2. придерживаться логики 1 User -> 1 Team

Так как операция создания пулл-реквеста требует назначения ревьюеров из команды пользователя, то в случае первого сценария будет неясно из какой именно команды назначать ревьюеров. Поэтому я принял решение пойти по второму сценарию. 

В таком случае после выполнения запросов* пользователь u1 покинет команду t1 и приcоединиться к команде t2.
